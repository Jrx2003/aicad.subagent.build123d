2. 一旦成功产出 artifact，主循环直接停止
3. `summary.last_error` 会明确写为：

### 问题 E3：annular groove 的 validator false negative 已被压到 tool/validator 层解决

这轮修复：

1. `service.py` 中 annular groove fallback 改成支持歧义锚点多候选：
   - `top_edge`
   - `center`
2. `_match_snapshot_axis_window_to_requirement_height(...)` 允许返回：
   - `world_space:center`
   - `world_space:top_edge`
   - `bbox_min_normalized:center`
   - `bbox_min_normalized:top_edge`
3. 文档和 system record 同步加入：
   - `ambiguous_annular_height_anchor_matching`

结果：

1. `benchmark/runs/20260405_223900/L1_218` 已通过

## sampled L2 首轮反馈：binary artifact 进入 round 2 prompt

在把 sampled L2 真正切到 V2 之后，第一轮 benchmark 很快暴露了一个明确的架构级 bug。

### 现象

1. `L2_63` 在 round 1 已经成功执行 `apply_cad_action`
2. `outputs/model.step` 和 `geometry_info.json` 已产出
3. 但在进入 round 2 前，runtime 直接崩溃

报错位置：

1. `src/sub_agent_runtime/context_manager.py`
2. `json.dumps(payload, ensure_ascii=False, indent=2)`

错误：

1. `TypeError: Object of type bytes is not JSON serializable`

### 根因

这不是某个单独 case 的 prompt 问题，而是新的 V2 prompt surface 在处理 write payload 时的通用序列化缺口：

1. `apply_cad_action` 的 `latest_write_payload` 里带有 `output_file_contents`
2. 其中包含真实 binary bytes
3. `context_manager` 会把 `latest_write_payload` 摘进 `latest_write_summary`
4. `compact_jsonish()` 之前只会截断字符串，不会把 `bytes` 转成 JSON-safe marker
5. 因此在第二轮 prompt 渲染时直接炸掉

### 修复

已修改：

1. `src/sub_agent_runtime/compact.py`
2. `tests/unit/sub_agent_runtime/test_v2_runtime.py`

修复方式：

1. 在 `compact_jsonish()` 的底层 `_clip_string()` 中统一把：
   - `bytes`
   - `bytearray`
   压成：
   - `"<N bytes>"`

这样做的好处是：

1. 修的是 compaction / prompt-safety 这一层，而不是在 `context_manager` 某个字段上单独打补丁
2. 后续不论 binary bytes 是来自：
   - `latest_write_summary`
   - 其他 compacted payload
   都会统一变成 JSON-safe

2. `feature_annular_groove` / `feature_revolved_groove_*`
   - `_snapshot_has_requirement_aligned_axisymmetric_groove()` 现在只按绝对轴向高度匹配
   - 但 `execute_cadquery` whole-part rebuild 常见的是整体居中建模
   - 应接受“按 bbox_min 归一化后的局部轴向窗口”匹配，而不是只认绝对世界坐标
