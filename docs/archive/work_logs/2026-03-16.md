## 代码架构迁移（Action-based -> Code-based）

### 新增/调整目标

1. 将 CAD 编辑动作统一为 `execute_code`，由 LLM 直接生成 CadQuery Python 片段执行。
2. 保留元操作（`rollback` / `clear_session`）与独立查询工具（`query_geometry` / `render_view` / `validate_requirement`）。
3. 保持可追溯与可重放：历史步骤继续完整保留在 `actions/`、`prompts/`、`plans/`、`queries/`、`outputs/`。

### 本次定位到的核心问题

1. 旧架构中 `_action_to_code` 扩展成本高，每新增几何能力都要新增 action 类型与转换逻辑。
2. 旧 action 枚举导致 planner 与执行层耦合过深，难以覆盖长尾 CAD 操作。
3. 历史重放依赖 action 翻译器，导致行为演化和兼容维护复杂。

### 本次完成的关键改动

1. `contracts.py`
   - `CADActionType` 收敛为：`execute_code` / `query_geometry` / `rollback` / `render_view` / `validate_requirement` / `clear_session`。
   - 删除几何微动作类型（`create_sketch`、`extrude` 等）。
   - `CADActionInput.action_type` 改为字符串输入并标准化，支持识别 legacy action 并在 service 返回升级提示。

2. `service.py`
   - `apply_cad_action` 主路径改为只执行 `execute_code`。
   - 删除几何 action->code 转换器路径，改为“历史代码片段累积重放”。
   - `rollback` 改为真实可用：按目标 step 截断历史并重放代码恢复状态。
   - 保留 `query_geometry/render_view/validate_requirement` 的元语义，但明确提示使用独立 tool。
   - requirement feature 校验改为从 `execute_code` 代码片段中做模式证据匹配（如 `.hole(` / `.fillet(` / `.chamfer(` / array API）。

3. `codegen.py`
   - `_SUPPORTED_ACTION_TYPES` 更新为 code-based 集合。
   - prompt 规则改为默认输出 `execute_code`，移除几何微动作约束。
   - reconstructed code 改为从历史 `execute_code` 片段拼接，而非 action 逆向映射。

4. `prompts/codegen_action.md`
   - 完整重写为 code-based planner 说明。
   - 增加 `execute_code` 严格输出格式和 CadQuery 常见模式模板。

5. 文档同步
   - 更新 `docs/cad_iteration/SYSTEM_RECORD.json`（版本升至 `1.5.0`，收敛到 execute_code 约束）。
   - 更新 `DESIGN_INTENT.md`、`ITERATION_PROTOCOL.md`、`TOOL_SURFACE.md` 的相应条目。
   - 更新 `CODEX.md`（英文）记录新目标与最新验证。
