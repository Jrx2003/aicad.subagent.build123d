## 今日新增根因：`L2_90` 的六孔圆盘失败不是 pattern 不会做，而是 guide circle 没被当成 guide

### 1. 失败现象

旧 full run 里，`L2_90` 的 planner 实际动作顺序并不离谱：

1. front datum plane 建直径 `100` 的圆盘底体
2. 在端面 sketch 上画一个 `70` 的 construction / pitch circle
3. 再画 6 个直径 `15` 的孔
4. 最后 `cut_extrude`

真正的底层问题不是 pattern 逻辑，而是 runtime 之前把 face-attached mixed-circle window 当成“不能直接 lower”的模糊 cut profile：

1. construction guide circle 和真实 hole circles 被一起看成普通 profile
2. `cut_extrude` 没有 lower 到 direct `hole`
3. 导致 face-attached subtractive replay 漂移，体积和 face family 都错

### 2. 第二层问题

我第一次把它修到 `hole` lowering 后，又暴露出第二个共享问题：

1. lowering 时如果把旧 `create_sketch.face_ref` 一起塞回新的 `hole`
2. 这个 `face_ref` 对当前 sketch window 已经是旧 step-local ref
3. 执行层会报 `invalid_reference: stale face_ref`

这说明对于“同一个 face-attached sketch window 内的 direct-hole lowering”：

1. 旧 `face_ref` 不能被当成稳定指针继续重放
2. 应该优先依赖当前 sketch face 上下文，最多补 fresh `_face_candidate_hint`
