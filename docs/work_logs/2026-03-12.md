## Additional Updates (Late Session)

1. Fixed `create_sketch` state semantics:
   - when solid already exists, `create_sketch` now stores sketch context without replacing the solid result.
   - prevents terminal `create_sketch` from collapsing final geometry to empty state.
2. Added runner solid-bootstrap execution policy:
   - in one-action mode, if no solid exists and planner already proposed a future solid-building action (`extrude`/`revolve`/`loft`), runtime executes actions up to that step in the same round.
   - event visibility: `trace/events.jsonl` -> `action_execution_policy`.
3. Expanded `add_polygon` compatibility:
   - supports explicit `points` polyline mode.
   - supports regular polygon params (`sides`/`side_count`, `radius`/`radius_outer`, optional `radius_inner`, `center`).
   - supports stepped profile params (`lengths` + `radiuses`/`radii`) for revolve-oriented inputs.
4. Hardened sketch action persistence:
   - `add_rectangle` / `add_circle` / `add_polygon` no longer wipe existing solids while sketching on top of a solid.
   - `extrude` / `revolve` / `cut_extrude` consume `_aicad_sketch` when present and then clear it.
