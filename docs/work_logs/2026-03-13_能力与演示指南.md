# 迭代式 CAD SubAgent：当前能力与终端演示指南（2026-03-13）

## 1. 当前已有能力（面向评估/演示）

1. 迭代生成闭环：
   - 支持按轮次执行 `plan -> action -> query -> verify`。
   - 每轮证据完整落盘：`prompts/`、`plans/`、`actions/`、`queries/`、`outputs/`、`summary.json`。
2. CAD 状态“可查询对象化”：
   - `query_geometry` 输出结构化实体窗口（solids/faces/edges）和分页偏移信息。
   - 可基于 `entity_ids` 做局部窗口检查，减少无关上下文。
3. 可控渲染能力：
   - `render_view` 支持自由相机参数（azimuth/elevation/zoom）。
   - 支持局部聚焦（`target_entity_ids`、`focus_center`、`focus_span`）。
4. 工具参数兼容性增强：
   - `extrude.both_sides`（对称拉伸）
   - `add_circle.position/center/diameter`
   - `create_sketch` 的 plane 别名与位置映射
   - `add_polygon.length_list/radius_list`（台阶轴类轮廓）
5. 运行时稳健改写（防止计划漂移）：
   - 不相关装饰特征自动抑制（如未要求时的 fillet/chamfer）。
   - 减材语义增强（如布尔差集场景下的 `extrude -> cut_extrude`）。
   - 中心阵列坐标归一（避免角点坐标误用）。
6. Benchmark 自动评分（无 LLM）：
   - 使用 STEP 几何签名评估（体积、bbox、面边数、质心/惯量等）。
   - 输出生成结果与真值的四视角预览，便于人工快速对比。

## 2. 终端演示（推荐顺序）

以下命令均在仓库根目录执行：`/Users/jerryx/code/aicad.subagent.iteration`

## 3. 运行结果如何检查

1. 过程证据：
   - `prompts/round_XX_request.json`
   - `plans/round_XX_response.json`
   - `queries/round_XX_*.json`
   - `trace/events.jsonl`
2. 产物：
   - `outputs/final_model.step`
   - `outputs/*preview*.png` 或 `outputs/*render_view*.png`
3. 评估：
   - `evaluation/benchmark_eval.json`
   - `evaluation/benchmark_eval_summary.txt`

## 4. 当前演示注意事项

1. 若 LLM 提示 `429` 或 `quota`，说明是上游模型账户额度问题，不是本地流程故障。
2. 若要做“无 LLM”链路检查，可使用：

```bash
./scripts/run_stage1_manual_probe.sh
```

该命令可验证沙箱动作执行、查询、渲染与产物落盘链路。
