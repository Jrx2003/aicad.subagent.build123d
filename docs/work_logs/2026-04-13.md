### 6. Build123d workplane / semi-profile contract hardening

新一轮真实 probe 暴露的不是 case-specific 几何细节，而是三类共享 Build123d contract 缺口：

1. directional drilling 场景里，模型会把 `Plane.offset(...)` 当成平面内坐标编码，而不是法向平移。
2. 工作平面姿态会被错误写成 `Plane.rotated(..., (0, 0, 0))`，把第二个参数误当 origin。
3. half-shell / split-shell profile 会猜错不存在或不兼容的 API：
   - `Circle(..., arc_size=...)`
   - `Semicircle(...)`

本轮已落地的共享修复：

1. `src/sub_agent/prompts/codegen.md`
   - 明确 `Plane.offset(...)` 只沿平面法向平移
   - 明确 `Plane.rotated(rotation, ordering=...)` 只改方向，不改 origin
   - 明确 `Circle(radius)` 永远是 full circle，不能用 `arc_size=...`
   - 明确没有 `Semicircle(...)` helper，应改用 `CenterArc(...)` / `RadiusArc(...)`
   - 对 explicit-radius half-shell 增加首选 first-pass recipe：
     - same-builder `Cylinder(...)`
     - `mode=Mode.SUBTRACT`
     - `mode=Mode.INTERSECT`
2. `src/sub_agent_runtime/skill_pack.py`
   - `execute_build123d_minimal_script_hygiene`
   - `positioned_holes_on_face_workplanes`
   - `half_shell_profile_from_semicircle_section`
   - 三处同步 workplane 法向、plane rotation、semi-profile contract 与 lower-risk half-shell recipe
3. `src/sub_agent_runtime/tool_runtime.py`
   - 新增 preflight lint:
     - `invalid_build123d_api.plane_rotated_origin_guess`
     - `invalid_build123d_keyword.circle_arc_size`
     - `invalid_build123d_api.semicircle_helper_name`
   - 新增 repair recipe:
     - `build123d_plane_rotation_contract`
     - `build123d_arc_profile_contract`
4. `docs/cad_iteration/TOOL_SURFACE.md`
5. `docs/cad_iteration/ITERATION_PROTOCOL.md`
6. `docs/cad_iteration/SYSTEM_RECORD.json`
   - `record_version` 更新到 `3.1.18`
